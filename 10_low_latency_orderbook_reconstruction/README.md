# 低延迟订单簿重建与市场微观结构分析

## 项目分类
- **类别**: 量化开发 / 高性能计算
- **对应岗位**: 量化开发工程师、高频交易工程师、低延迟系统工程师

## 项目定位
从 Level2 行情数据实时重建订单簿，分析市场微观结构特征（订单流不平衡、做市商行为、大单拆分检测）。

## 核心创新点
1. **高效订单簿重建**: 基于事件驱动的订单簿增量更新算法
2. **订单流分析**: 检测大单拆分、冰山订单、 spoofing 等微观结构信号
3. **延迟优化**: 全链路延迟分析，识别瓶颈并优化

## 技术栈
Python, NumPy, Cython/C++(可选), Redis, WebSocket

## 关键挑战
- 如何处理丢失或乱序的行情数据
- 如何在 Python 中实现接近 C++ 的性能（或使用混合方案）
- 如何从微观结构信号中提取可交易的 Alpha

## 预期成果
- 订单簿重建引擎（支持深圳/上海 Level2）
- 订单流分析工具（大单检测、买卖压力指标）
- 延迟监控 Dashboard
- 微观结构因子研究报告

## 简历表述示例
"实现 Level2 订单簿重建引擎，支持 10ms 级延迟的实时更新，设计订单流不平衡指标，在 XX 股票上测试显示该指标对未来 YY 分钟收益有显著预测能力（IC=ZZ）"
